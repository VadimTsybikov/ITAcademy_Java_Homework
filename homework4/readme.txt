Ханойские башни.

Сценарий

Выводится меню, в котором нужно выбрать игру вручную или автоматическое решение.
В любом случае инициализируется игровое поле: кольца расположены на 1-м штыре, самое широкое кольцо
внизу, самое узкое - вверху.
Игровое поле выводится на экран.
Если выбрана игра вручную, пользователь вводит количество колец на первом штыре.
Пользователю предлагается ввести два числа: номер исходного штыря (с которого снимается кольцо),
и номер целевого штыря (на который оно переходит).
При этом проверяется:
- есть ли кольца на исходном штыре;
- не кладется ли более широкое кольцо на более узкое.
Если какое-то условие не соблюдатеся - пользователя выдается соответствующее сообщение и игра
продолжается без учета этого хода.
Если шаг допустим - происходит перемещение кольца и выводится актуальное состояние игрового поля.
Запросы на перемещение колец происходят до тех пор, пока на последнем штыре не окажутся все кольца,
и при этом они расположены по возрастанию веса (ширины).
Если выбрано автоматическое решение, происходит пошаговое решение с выводом состояния поля
при каждом перемещении.

Технически

Точка входа в программу находится в классе "TowerOfHanoiMain".
В двумерном массиве нижние кольца i-го штыря находятся в позиции [0][i].
Параллельно с перемещением колец ведется одномерный массив, в котором сохраняются индексы
верхних колец на штырях. Массив инициализируется значениями -1.
Автоматичесое решение нагло стырено с Хабра =).
